# [Spike]: [Concise Title]

A spike ticket for time-boxed investigation. Deliverable is findings and a recommendation, not implementation.

## Summary

What we need to learn or decide. One to two sentences.

## Research Question / Goal

- What are we trying to determine? (feasibility, approach, scope, effort)
- What decision will this unblock?

## Scope of Investigation

- Systems, code areas, or docs to explore
- Out of scope (what we are not investigating)

## Deliverables

- [ ] Findings documented (spike doc or Confluence)
- [ ] Recommendation or options with pros/cons
- [ ] Follow-up tickets identified (if any)

## Context

Why this spike now. What triggered it (project, bug, design question).

## Tech Specs (optional)

- Entry point: [file or doc](github-link) - where to start looking
- Key areas to investigate
- Constraint or gotcha (e.g. "Do not modify production config")

## Open Questions (optional)

- Questions to resolve during the spike or with stakeholders
- Format as list; can be updated with answers as the spike progresses

## Additional Context (optional)

- Related tickets, PRs, or design docs
- Links to prior discussions or spike docs
